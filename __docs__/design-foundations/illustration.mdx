# Illustration

Khan Academy's illustration system is built entirely on shapes.  From Khanmigo to empty states to confetti blasts — every illustration element is shape based, used at different levels of meaning, fidelity, and emotion.

## Audience calibration

The same system serves multiple audiences. The principles don't change — the emotional register does.

### Learner side

Playful, vibrant, character-driven. Khanmigo is central. Customization makes the experience personal and ownable. Celebration moments are high-energy — confetti, expressive Khanmigo states, reward icons. Saturation is high.

<img src={audienceCalibration1Img} alt="Audience calibration 1" />

### Teacher/Admin side

Warmer and more restrained. Illustrations appear as moments of joy in a data-dense interface — their job is to prevent the product from feeling cold. Spot illustrations should feel reassuring, not playful. Saturation is moderate.

<img src={audienceCalibration2Img} alt="Audience calibration 2" />

## Usage guidelines

Guidance for how to apply illustration correctly across contexts. Visual examples will be defined here as the system matures.

### Contrast by meaning

Ask first: if this illustration were removed, would the user lose meaning? If yes, it's meaningful — apply 3:1.
Khanmigo’s body color requires 3:1 against its background. Accessories do not.
Confetti and accent shapes are always decorative — no hard minimum, but avoid combinations that feel unintentional.

### Flat and geometric

Never apply gradients, drop shadows, or dimensional shading to illustration elements.
Corners and edges should feel intentional — rounded only where the form calls for it.
Complexity should come from composition, not from surface treatment.

### Color from primitives

All illustration colors reference shared primitive tokens — never introduce custom hex values.
Lean more saturated for learner-facing illustration; more restrained for teacher-facing contexts.
Khanmigo’s body color is the learner's identity color — use it consistently wherever Khanmigo appears for that learner.

## Types of illustration

Each tier has a distinct purpose, audience, and contrast requirement. Identifying the correct tier is the first step in any illustration decision.

### Khanmigo

Khanmig’s body is a hexagon. Facial features, body color, accessories, and props are shapes composed together to create expression and personality. Body color is the learner's identity color. Accessories are decorative.
See Khanmigo and co. for more

<img src={typesOfIllustration1Img} alt="Types of illustration 1" />

### UI illustration

Small illustrations embedded directly in the interface — gem icons, reward indicators, progress markers. These are always meaningful and always require 3:1 contrast against their background.

<img src={typesOfIllustration2Img} alt="Types of illustration 2" />

### Spot illustration

Empty states and contextual illustrations composed of shapes to convey meaning in moments where data or content is absent. Yet, they are always paired with text that carries the same meaning, making them decorative — no contrast minimum, though mindfulness is always encouraged.

<img src={typesOfIllustration3Img} alt="Types of illustration 3" />

### Scene

Mission maps and themed worlds are built from flat geometric shapes. Trees, campfires, soccer fields — all shapes arranged to create a sense of place. Assess each element individually for contrast requirements.

<img src={typesOfIllustration4Img} alt="Types of illustration 4" />

### Accent

Floating shapes, confetti blasts, and celebration scatter elements used purely for delight and energy. Always decorative — no contrast minimum, though mindfulness is always encouraged.

<img src={typesOfIllustration5Img} alt="Types of illustration 5" />

## Accessibility

The key question: if this illustration were removed, would the user lose meaning? If yes — apply 3:1.

| Type | Meaningful | Decorative | Contextual |
| --- | --- | --- | --- |
| Contrast requirement | 3:1 | No requirement | Assess |
| Rationale | The illustration communicates something the user must understand. Examples: gem icon in the score counter, class identity shape, Khamigo’s body color against its background. | The illustration adds delight but carries no meaning the user must interpret. No hard minimum — mindfulness is always encouraged. Examples: Khamigo accessories, confetti shapes, floating accent elements. | Partly meaningful, partly decorative — assess each element individually. Example: In a mission map, Khamigo’s body is meaningful (3:1 required); the ranger hat is decorative (no minimum); path waypoints are navigational and meaningful (3:1 required). |

## Roles and responsibilities

Illustration is a craft and a product decision simultaneously — where it appears and what it looks like are inseparable. Visual designers own both.

| Responsibility | Visual design | Product design |
| --- | --- | --- |
| Create all illustration assets | Owns | – |
| Determine whether a moment needs illustration | Owns | – |
| Decide which tier and emotional register is appropriate | Owns | – |
| Ensure new illustration fits the existing system | Owns | – |
| Flag moments that might benefit from illustration | Contributes | Contributes |
